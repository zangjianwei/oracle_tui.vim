1. Introduction                              *oracle_tui-introduction*

Oracle TUI is a lightweight, terminal-based Oracle UNIX/Linux database
management client built for Vim. Packed with features, it provides a data
editing experience comparable to GUI tools.

Features:
- Lightning-fast Connection: Supports auto-login via environment variables,
  connect to your database with a single command
- Spreadsheet-like Editing: Edit data in Vim just like Excel
  (SELECT ... FOR UPDATE)
- Keyboard-Centric Workflow: F8 to execute SQL, F12 to submit data changes
- Safety Mechanisms: Forces an error and intercepts exit if there are
  uncommitted transactions when quitting Vim
- Smart Completion: Supports table name auto-completion (including
  non-prefix strings)
- Multi-Statement Support: Supports PL/SQL block execution with separated
  result displays
- Three Window Modes: SQL Execution Window, Query Window, Data Modification
  Window

2. Installation                              *oracle_tui-install*

2.1 Dependencies                            *oracle_tui-deps*
- Linux/Unix environment
- Oracle Instant Client (with sqlplus configured)
- Vim 7.4 or higher, Vim 8.2 or higher recommended

2.2 Setup Steps                             *oracle_tui-setup*

1. Place oracle_tui.tar in the installation user's $HOME directory:
   tar xvf oracle_tui.tar

2. Modify profile, add PATH:
   export PATH=$PATH:$HOME/oracle_tui:.

3. Check if NLS_LANG and LC_CTYPE environment variables are UTF-8.
   If not, set the following variables:
   export NLS_LANG=AMERICAN_AMERICA.AL32UTF8
   export LC_CTYPE=en_US.UTF-8

   If you prefer not to modify the above environment variables, set these
   instead:
   export TUI_NLS_LANG=AMERICAN_AMERICA.AL32UTF8
   export TUI_LC_CTYPE=en_US.UTF-8

   Note:Please select the correct [language]_[region]
4. Set file encoding in ~/.vimrc or your own .vimrc:
   set encoding=utf-8
   set fileencodings=ucs-bom,utf-8,gb18030,gbk,gb2312,cp936,latin1
   set termencoding=utf-8

5. Set DBUSER and DBPASS environment variables (optional, for auto-login):
   export DBUSER=your_username
   export DBPASS=your_password

6. Set alias (optional, for auto-connecting to database):
   alias vidb='vim -c "call oracle_tui_start#ConnectDB()"'

   Or specify vimrc:
   alias vidb='vim -u /path/to/your/.vimrc -c "call oracle_tui_start#ConnectDB()"'

   Or specify file encoding (no need to set encoding in .vimrc):
   alias vidb='vim --cmd "set encoding=utf-8|set fileencodings=ucs-bom,utf-8,gb18030,gbk,gb2312,cp936,latin1|set termencoding=utf-8" -c "call oracle_tui_start#ConnectDB()"'

7. Install help file (optional):
   vim
   :helptags ~/.vim/doc

8. Source profile:
   . ~/.bash_profile
   or
   . ~/.profile

3. Startup                                   *oracle_tui-start*

3.1 Terminal Settings

Make sure your terminal is set to UTF-8 encoding before starting.

3.2 Startup Methods

Method 1: Connect manually in Vim
  :Connect
- If DBUSER and DBPASS environment variables are set, auto-login is performed
- Otherwise, you will be prompted for username and password

Force manual username and password entry:
  :Connect -u

Method 2: Start with alias
  vidb <filename>

- Keyboard shortcuts become active only after successful connection

4. Window Modes                              *oracle_tui-windows*

Oracle TUI provides three window modes:

4.1 SQL Execution Window                   *oracle_tui-window-sql*
    The default window when entering Vim.
    Write and execute SQL statements in this window.
    - Execute SQL
    - Object navigation
    - Auto-completion
    - View object definitions
    - View execution plans

4.2 Query Window                           *oracle_tui-window-query*
    Entered when executing a single SELECT statement.
    - Fixed header row
    - Full column headers display, overcoming sqlplus truncation limits
    - Column sorting
    - Shorten, move, and cut columns
    - Filter data within columns
    - Column summation

4.3 Data Modification Window               *oracle_tui-window-edit*
    Entered when executing SELECT ... FOR UPDATE.
    - Spreadsheet-style editing
    - Fixed header row
    - Auto-alignment during insert/delete/modify/replace operations
    - Numeric fields are right-aligned
    - VARCHAR2 fields intelligently recognize trailing spaces
    - CLOB/BLOB fields: generates a file for each field and displays filename
    - Replacement operations preserve headers and CLOB/BLOB filenames
    - DATE/TIMESTAMP fields: auto-prompt for input format when cursor lands
    - CLOB/BLOB fields: auto-prompt for Ctrl+A to edit

5. Keybindings Quick Reference               *oracle_tui-keybindings*

5.1 SQL Execution Window                   *oracle_tui-keys-sql*

   Key       Function
   ------    --------
   F1        Show help
   F8        Execute SQL (Visual mode executes selection, Normal mode
             executes current line)
   Ctrl+C    Interrupt current operation
   F2        Rollback transaction
   F6        Commit transaction
   F4        Check for uncommitted transactions
   F5        View SQL execution plan
   F7        List database objects
             F9 and F10 work in the popup window
   F9        Show table structure description for the table under cursor
   F10       Show DDL statement for the object under cursor
   -         Decrease current window width
   =         Increase current window width
   Shift+-   Decrease current window height
   Shift+=   Increase current window height
   Ctrl+↑    Jump to upper window
   Ctrl+↓    Jump to lower window
   Ctrl+→    Jump to right window
   Ctrl+←    Jump to left window
   Ctrl+n    Object name auto-complete (prefix match, Insert mode)
   Ctrl+k    Table name auto-complete (supports middle-string match,
             Insert/Normal mode)
             F9 and F10 work in the popup window
   gt        Switch between tabs

6. Security Mechanisms                       *oracle_tui-security*

- Forces an error and intercepts exit if there are uncommitted transactions
  when quitting Vim
- Replacement operations in Data Modification Window do not replace header row
- Replacement operations do not replace CLOB/BLOB filenames

7. Limitations                              *oracle_tui-limitations*

- Does not support modification of tables with LONG or LONG RAW field types
- AIX Vim 9 has a bug with list processing; Vim 9 cannot be used on AIX
- Vim 7.4 limitations:
  In Data Modification Window:
  1. Cannot execute zl, zh, zL, zH commands 
  2. Tab characters display as   during editing
  3. Cannot split window vertically

8. FAQ                                       *oracle_tui-faq*

Q: Connection failed?
A: Check if ORACLE_HOME and PATH are correctly set, and ensure sqlplus is
   available.

Q: Chinese characters display garbled?
A: Check that encoding, fileencodings, and termencoding are correctly set.
   Check that NLS_LANG and LC_CTYPE are correctly set.

Q: How to exit Data Modification Window?
A: Use :q to close the window. You will be prompted if there are uncommitted
   changes.

Q: Auto-completion not working?
A: Ensure you are properly connected to the database and the object name
   exists.
